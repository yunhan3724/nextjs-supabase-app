## 1. 문제 `Problem`

> 해결하려는 핵심 문제 3가지

- **분산된 운영 도구**: 주최자가 카카오톡·엑셀·머릿속에 공지, 참석 체크, 카풀 조율, 정산을 나눠서 처리한다
- **반복되는 수작업**: 매 모임마다 참석 여부를 일일이 물어보고, 카풀을 DM으로 매칭하고, 계산기로 정산한다
- **정보 파편화**: 참여자 입장에서 일정·장소·정산 정보가 여러 채팅방에 흩어져 있어 찾기 어렵다

### 기존 대안

| 현재 방식            | 문제점                                         |
| -------------------- | ---------------------------------------------- |
| 단체 카카오톡 채팅방 | 공지가 메시지에 묻히고, 참석 여부 집계가 안 됨 |
| 엑셀/스프레드시트    | 실시간 공유가 어렵고, 모바일 접근성 낮음       |
| 네이버 밴드 / 카페   | 가입 장벽, 기능 과다, 정산/카풀 기능 없음      |

## 2. 고객 세그먼트 `Customer Segments`

> 누구의 문제를 푸는가

### 얼리어답터 (MVP 집중 타깃)

- **모임 총무·주최자**: 수영·헬스·테니스 동호회, 친구 모임 등을 정기 또는 비정기로 운영하는 20~40대. 참석 관리·정산에 매번 시간이 소요되어 불편함을 느끼는 사람
- **자주 모이는 소규모 그룹 멤버**: 링크 하나로 일정 확인·참석 응답·카풀 신청을 빠르게 마치고 싶은 참여자

### 페르소나

| 페르소나                | 특징                              | 핵심 니즈                                             |
| ----------------------- | --------------------------------- | ----------------------------------------------------- |
| **민준 (주최자, 30대)** | 수영 동호회 총무, 매달 1~2회 모임 | 링크 하나 보내면 참석·카풀·정산이 자동으로 굴러갔으면 |
| **지은 (참여자, 20대)** | 친구 모임에 가끔 참여             | 복잡한 가입 없이 링크 클릭 → 바로 정보 확인·응답      |

## 3. 가치 제안 `Unique Value Proposition`

> 왜 이 제품이어야 하는가

### 핵심 가치

**"초대 링크 하나로, 주최자는 모임 운영을 자동화하고 참여자는 모든 정보를 한 곳에서 본다"**

### 주최자 가치

- 이벤트 생성 → 링크 공유 → 공지·참석·카풀·정산을 **한 화면**에서 끝냄
- 반복 수작업(참석 체크, 정산 계산, 카풀 매칭)을 **앱이 대신 처리**

### 참여자 가치

- 별도 앱 설치·복잡한 가입 없이 **링크 클릭 → 구글 로그인 → 즉시 참여**
- 일정·공지·카풀·내 정산을 **한 곳**에서 확인

### 상위 개념

> 모임 주최자를 위한 올인원 이벤트 운영 도우미

## 4. 솔루션 `Solution`

> 문제를 어떻게 풀 것인가

| 문제             | 솔루션                                                                       |
| ---------------- | ---------------------------------------------------------------------------- |
| 분산된 운영 도구 | **이벤트 단위 통합 페이지** — 공지·RSVP·카풀·정산을 탭 하나로 모음           |
| 반복되는 수작업  | **자동화 기능** — RSVP 자동 집계, n분의1 정산 자동 계산, 카풀 신청/확정 흐름 |
| 정보 파편화      | **초대 링크** — 토큰 기반 링크 하나로 로그인 → 이벤트 참여 → 전체 정보 접근  |

### 핵심 기능 (F001~F006)

- **F001** 이벤트 생성·관리 (draft→published→closed)
- **F002** 공지 CRUD + 핀 고정
- **F003** 참여자 RSVP (going/not_going/maybe) + 동반 인원
- **F004** 초대 링크 (토큰 발급·만료·최대 사용 횟수)
- **F005** 카풀 (차량 등록·탑승 신청·확정·잔여석)
- **F006** 정산 (n분의1 자동 분배·송금 현황 추적)

## 5. 채널 `Channels`

> 고객에게 어떻게 도달할 것인가

### 획득 채널

- **바이럴 (핵심)**: 초대 링크 자체가 채널. 참여자가 링크를 받아 서비스를 경험 → 본인이 모임을 열 때 재사용
- **입소문**: 동호회·소모임 커뮤니티(네이버 카페, 오픈채팅방) 에서 주최자 추천
- **SNS/콘텐츠**: "모임 총무를 위한 툴" 포지셔닝으로 인스타그램·블로그 타깃 콘텐츠

### 활성화/유지 채널

- 이메일 인증 (Supabase Auth 내장)
- 이벤트 생성 완료 후 링크 공유 유도 (앱 내 CTA)

## 6. 수익 흐름 `Revenue Streams`

> MVP 단계에서는 수익 모델을 단순하게 유지

| 단계         | 모델                 | 내용                                                                         |
| ------------ | -------------------- | ---------------------------------------------------------------------------- |
| **MVP**      | 무료                 | 핵심 기능 전체 무료. 사용자 확보·피드백 수집 우선                            |
| **Post-MVP** | 프리미엄(추후 검토)  | 이벤트 개수 제한 해제, 반복 이벤트, 고급 정산(가중치 분배), 커스텀 도메인 등 |
| **장기**     | B2B / 팀 플랜 (검토) | 동호회·소모임 단체 대상 월정액                                               |

> MVP에서는 수익화보다 **제품-시장 적합성(PMF) 검증**에 집중한다.

## 7. 비용 구조 `Cost Structure`

> 초기 비용을 최소화하는 스타터 기반 아키텍처

| 항목            | 내용                                  | 비용 수준                              |
| --------------- | ------------------------------------- | -------------------------------------- |
| **Supabase**    | DB(Postgres) + Auth + RLS + Storage   | 무료 플랜으로 MVP 시작 가능            |
| **Vercel**      | Next.js 15 호스팅                     | 무료 플랜으로 시작, 트래픽 증가 시 Pro |
| **개발 인건비** | 솔로 개발자 / 소규모 팀               | 가장 큰 비용 항목                      |
| **도메인**      | 커스텀 도메인                         | 연간 소액                              |
| **마케팅**      | MVP 단계: 콘텐츠·커뮤니티 중심 (무료) | 최소화                                 |

> 인프라 초기 비용 거의 0. **개발 시간이 유일한 실질 비용**.

## 8. 핵심 지표 `Key Metrics`

> 제품이 올바른 방향으로 가고 있는지 판단하는 지표

| 지표                | 설명                                               | 목표 (MVP 초기)  |
| ------------------- | -------------------------------------------------- | ---------------- |
| **주최자 완주율**   | 이벤트 생성 → 초대 → 정산까지 완료한 주최자 비율   | 50% 이상         |
| **초대 전환율**     | 초대 링크 클릭 → 참여하기(redeem) 완료 비율        | 60% 이상         |
| **RSVP 응답률**     | 이벤트당 참여자 중 going/not_going/maybe 응답 비율 | 70% 이상         |
| **주최자 재사용률** | 2회 이상 이벤트를 생성한 주최자 비율               | 30% 이상         |
| **카풀 매칭률**     | 카풀 등록 이벤트에서 탑승 confirmed 비율           | 측정 시작        |
| **DAU/MAU**         | 서비스 전반 활성화 지표                            | MVP 후 추적 시작 |

## 9. 경쟁 우위 `Unfair Advantage`

> 쉽게 복사하거나 구매할 수 없는 차별점

- **초대 링크 UX**: 별도 앱 설치·가입 없이 링크 → 구글 로그인 → 즉시 참여. 진입 장벽 최소화
- **주최자 특화 올인원**: 공지·RSVP·카풀·정산을 하나의 이벤트 페이지에 통합. 모임 총무가 실제로 쓰는 기능만
- **모바일 우선 카드형 UI**: 카카오톡에 익숙한 사용자가 자연스럽게 사용할 수 있는 밝고 친근한 디자인
- **네트워크 효과**: 참여자가 링크를 통해 서비스를 경험 → 다음 모임에서 주최자로 전환되는 바이럴 구조
